Thiết kế & Kiến trúc Module EAM

Hệ thống EAM MES Package cung cấp 4 phân hệ lõi phục vụ giám sát thiết bị và vận hành nhà máy. Các thành phần được đóng gói dưới dạng Laravel Package chuẩn để cắm ghép vào dự án của doanh nghiệp.

Các Module Nghiệp vụ Lõi (Submodules)

  • Checklist (Kiểm tra thiết bị - src/Checklist/): Quản lý quy trình thiết lập các bộ câu hỏi kiểm tra, tạo phiên (session) thực thi cho nhân viên vận hành trước ca, và ghi nhận chi tiết (details) kết quả kiểm tra từng hạng mục.
  • Error Monitoring (Giám sát lỗi máy - src/ErrorMonitoring/): Theo dõi thời gian dừng máy (downtime), thu nhận thông tin mã lỗi và tự động phân tích các chỉ số quản lý thiết bị quan trọng như OEE, MTBF, MTTR.
  • Maintenance (Bảo trì vòng đời - src/Maintenance/): Cấu trúc danh mục bảo trì (categories), hạng mục chi tiết (items), quản lý kế hoạch tuần hoàn (plans), phân phối lịch trình (schedules) và ghi lại nhật ký thực thi (logs).
  • Parameter Log (Ghi nhận thông số sensor - src/ParameterLog/): Hệ thống lưu trữ dữ liệu chuỗi thời gian (Timeseries) thu thập các thông số vận hành của thiết bị như nhiệt độ, áp suất, điện áp... trực tiếp từ các sensor.
  • Equipment (Quản lý Equipment - src/Equipment/MasterData/): Lớp quản lý hoạt động và ghi log thời gian thực (IoT logs, seeding lỗi dừng ngắn) cho các thiết bị.
  • Masterdata Equipment (Dữ liệu gốc Thiết bị - src/Masterdata/Equipment/): Định nghĩa cấu trúc dữ liệu gốc của thiết bị (equipment), nhóm thiết bị (categories), thông số kỹ thuật (parameters), các ngưỡng tiêu chuẩn (standard parameters) và phân loại lỗi hệ thống (errors).

Quy trình Triển khai Package

Hướng dẫn chi tiết quy trình chèn thư viện vào hệ thống Laravel và publish tài nguyên tương ứng:

1
Cài đặt gói Package

Thêm khai báo repository và yêu cầu gói vào tệp composer.json của bạn:

composer.json 
{
    "repositories": [
        {
            "type": "vcs",
            "url": "https://github.com/LavioDev/eam-mes-package"
        }
    ],
    "require": {
        "laviodev/eam-mes-package": "dev-main"
    }
}

Sau đó chạy lệnh cập nhật để cài đặt:

Bash
composer update laviodev/eam-mes-package
2
Publish file cấu hình chính

File config chính sẽ được copy từ vendor sang đường dẫn config/eam.php để quản lý whitelist bảng, middleware và danh sách class mở rộng:

Bash
php artisan vendor:publish --tag="eam-mes-package-config"
3
Publish phân hệ Cốt lõi (Core Submodule)

Phân hệ Cốt lõi chứa bảng quản lý yêu cầu mở rộng Schema động (eamo_extension_requests). Đây là nền tảng bắt buộc để hệ thống vận hành và đồng bộ dữ liệu:

Bash
php artisan eam-mes:publish --submodule=core
4
Publish module Dữ liệu gốc Thiết bị (Masterdata Equipment Module)

Định nghĩa cấu trúc dữ liệu gốc của thiết bị (bảng equipment, equipment_categories, equipment_parameters, standard_parameters, và equipment_errors) giúp thiết lập nền tảng liên kết cho mọi nghiệp vụ vận hành:

Bash
php artisan eam-mes:publish --module=masterdata-equipment
5
Publish các Module nghiệp vụ khác & Submodule

Bạn có thể publish toàn bộ các module và submodule cùng lúc, hoặc lựa chọn cụ thể theo nhu cầu của dự án:

Bash
# Publish TOÀN BỘ các module và submodule cùng lúc php artisan eam-mes:publish --all # Hoặc publish toàn bộ module Equipment (bao gồm tất cả các submodule bên dưới) php artisan eam-mes:publish --module=equipment # Hoặc chỉ publish từng submodule riêng lẻ của Equipment: # 1. Submodule Management (Quản lý Thiết bị - IoT Logs & Dừng ngắn) php artisan eam-mes:publish --submodule=management # 2. Submodule Checklist (Kiểm tra đầu ca/định kỳ) php artisan eam-mes:publish --submodule=checklist # 3. Submodule Error Monitoring (Giám sát lỗi & OEE) php artisan eam-mes:publish --submodule=error-monitoring # 4. Submodule Maintenance (Kế hoạch & Nhật ký bảo trì) php artisan eam-mes:publish --submodule=maintenance # 5. Submodule Parameter Log (Ghi thông số cảm biến) php artisan eam-mes:publish --submodule=parameter-log
6
Thực thi migrate cơ sở dữ liệu

Tạo các bảng cơ sở dữ liệu eamo_* trong DB:

Bash
php artisan migrate

Mở rộng Schema qua Class PHP

Cơ chế này giải quyết bài toán: Tùy biến thêm trường dữ liệu động cho các bảng dữ liệu lõi của package mà không cần sửa trực tiếp mã nguồn lõi. Nhà phát triển chỉ cần khai báo mô tả cấu trúc trường thông qua một Class PHP.

Nguyên lý hoạt động

  • Mọi thay đổi cấu trúc DB đều phải thực thi qua file **Migration**. Hệ thống **không bao giờ thực thi ALTER TABLE trực tiếp** trên DB để tránh race conditions và mất audit trail.
  • Sử dụng Class mô tả dạng Value Object (định hình bằng class ColumnDefinition) giúp hệ thống tự động suy diễn cấu trúc để sinh cả phương thức up() (thêm cột) và down() (xóa cột khi rollback) hoàn toàn tự động.

Hướng dẫn chi tiết các bước triển khai

Bước 1: Khai báo Class mở rộng

Tạo một tệp PHP mới nằm trong thư mục app/Extensions/ và kế thừa interface Spatie\LaravelPackageTools\Contracts\TableExtension:

app/Extensions/MaintenancePlanExtension.php
namespace App\Extensions; use Spatie\LaravelPackageTools\Contracts\TableExtension; use Spatie\LaravelPackageTools\Extensions\ColumnDefinition; class MaintenancePlanExtension implements TableExtension { // Xác định bảng đích được phép mở rộng trong Whitelist public function targetTable(): string { return 'eamo_maintenance_plans'; } // Định nghĩa danh sách các trường muốn bổ sung thêm public function columns(): array { return [ ColumnDefinition::make('department_id', 'string') ->length(36) ->nullable() ->after('user_id'), ColumnDefinition::make('is_urgent', 'boolean') ->default(false), ]; } // Thứ tự ưu tiên xử lý (nhỏ chạy trước) public function priority(): int { return 10; } }

Bước 2: Đăng ký Class trong cấu hình

Mở file config/eam.php và khai báo Class vừa tạo vào mảng extensions:

config/eam.php
// config/eam.php 'extensions' => [ App\Extensions\MaintenancePlanExtension::class, ],

Bước 3: Chạy Command đồng bộ DB

Sử dụng Artisan command của package để quét class cấu hình, kiểm tra trùng lặp và ghi file migration vật lý vào thư mục database/migrations/:

Artisan CLI
# Kiểm thử thay đổi cấu trúc bảng (không ghi file thực tế) php artisan eam:sync-extensions --dry-run # Sinh tệp tin migration vật lý và chạy migrate cập nhật DB lập tức php artisan eam:sync-extensions --migrate # Ép buộc sinh lại migration bỏ qua kiểm tra trùng lặp php artisan eam:sync-extensions --force

Vị trí tệp sinh ra & Naming Convention

Sau khi thực thi lệnh đồng bộ, hệ thống sẽ sinh ra file migration nằm ở thư mục database/migrations/ của host app dưới tên mẫu:

database/migrations/{Y_m_d_His}_extend_{table_name}_table.php

Ví dụ: database/migrations/2026_07_05_120000_extend_eamo_maintenance_plans_table.php

Mở rộng Schema qua API Endpoint

Trong các kiến trúc Microservices hoặc Headless, việc mở rộng cấu trúc bảng cơ sở dữ liệu có thể được trigger tự động từ một hệ thống quản lý tập trung bên ngoài thông qua API Endpoint.

Cơ chế hoạt động bất đồng bộ (Queue Job)

Để ngăn chặn lỗi khóa bảng (table lock), race condition khi có nhiều request đồng thời, package sử dụng cơ chế xử lý bất đồng bộ:

Quy trình xử lý yêu cầu mở rộng qua API
POST Request
Đón nhận & Xác thực whitelist của bảng
→
Queue Job
Ghi nhật ký request & Đẩy Job ngầm vào hàng đợi
→
Migration & Run
Worker tự động sinh file & thực thi migrate

Cách thức sử dụng API

1. Gửi yêu cầu thêm cột (POST)

Gửi payload dạng JSON mô tả các cột cần tạo mới:

POST /eam/api/extensions
POST /eam/api/extensions Content-Type: application/json { "table": "eamo_maintenance_plans", "columns": [ { "name": "department_id", "type": "string", "length": 36, "nullable": true, "after": "user_id" } ] }

Phản hồi thành công (HTTP Status Code: 202 Accepted): Trả về ngay lập tức mã yêu cầu (Request ID) cùng URL để kiểm tra trạng thái xử lý sau đó.

2. Kiểm tra trạng thái tiến trình (GET)

Gọi API để kiểm tra tiến trình xử lý dưới background:

GET /eam/api/extensions/{id}
GET /eam/api/extensions/42

Dữ liệu phản hồi mẫu:

JSON Response
{ "id": 42, "table": "eamo_maintenance_plans", "status": "done", "migration_file": "2026_07_05_120000_extend_eamo_maintenance_plans_table.php", "error_message": null }

Bảo mật & Hàng đợi cần kích hoạt

  • Queue Worker: Để xử lý các job sinh file dưới nền, cần chạy lệnh khởi động Queue Worker chuyên biệt:
    php artisan queue:work --queue=eam-extensions
  • Audit Trail: Mọi yêu cầu được ghi lại trong bảng nhật ký hệ thống: eamo_extension_requests.
  • Bảo vệ API: Khai báo middleware xác thực (Passport, Sanctum) cho endpoint này bằng cách cấu hình thuộc tính api_middleware trong file config/eam.php.
